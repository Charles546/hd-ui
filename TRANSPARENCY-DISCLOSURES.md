# Transparency Disclosures

Effective date: 2026-03-26

This document provides transparency information for Honeydipper Web UI and related GitHub App listing use cases.

## 1. Security Measures

The project is designed to support secure deployment practices, including:

- TLS/HTTPS for all external traffic
- role-based access control enforced by backend policies
- authentication support (Bearer token and Basic auth, depending on deployment)
- least-privilege operational model for integration credentials
- auditability through workflow/session status visibility
- dependency and source control protections through standard GitHub workflows and reviews

### Shared Responsibility

Security is a shared responsibility between software maintainers and deployers. Final security posture depends on environment configuration (network, secrets, IAM, host hardening, and monitoring).

## 2. Data Handling

### Data Processed

Depending on deployment configuration, the system may process:

- user identity/subject metadata for authorization
- repository, event, and workflow execution metadata
- integration payloads and operational status data
- optional logs and diagnostics generated by workflows

### Data Storage

- Browser-side credentials are stored in `sessionStorage` for active sessions.
- Persistent data retention is controlled by backend services and infrastructure configured by operators.
- This project does not include built-in ad tracking or profiling systems.

### Data Sharing

Data is shared only with configured systems required for operation, such as GitHub APIs, Honeydipper services, and explicitly configured integrations.

## 3. Compliance and Governance

### Compliance Position

No standalone certification is claimed by default for this repository alone. Compliance outcomes depend on deployment environment, operational controls, and organizational governance.

Potential frameworks deployers may align to include (as applicable):

- SOC 2 controls
- ISO/IEC 27001 controls
- GDPR/CCPA privacy requirements
- internal security and retention policies

### Governance Practices

- policy and legal documents are versioned in-repo
- changes are tracked through source control history
- open issue tracking is available for policy/security concerns

## 4. Incident and Vulnerability Reporting

If you discover a security issue or potential vulnerability, report it through the repository issue tracker (or private channel if configured by repository owners):

- https://github.com/Charles546/hd-ui/issues

Include reproduction details, impact assessment, and affected versions when possible.
